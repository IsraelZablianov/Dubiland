## Decisions
- For handbook reading expansion, define launch priority as one book per age group to avoid overfitting the first production wave to a single developmental band.
- Keep ladder specs auditable by binding each priority book to explicit nikud policy and sentence complexity constraints.
- Use paired FED + Content Writer delegation immediately after spec creation so no reading spec remains "not yet handed off."
- Treat reading-game delegation as multi-lane default: implementation + content/audio + mechanics review.
- Use issue search (`q=`) and direct identifier lookup (`/api/issues/{identifier}`) when reconciling existing work.
- For handbook literacy checkpoints, keep validation action-based in-page (no detached quiz screen) so decoding practice stays inside story context.
- Keep nikud fully explicit through Level 1-2 handbook checkpoints and fade only in Level 3 bridge tasks.
- In quality-overhaul heartbeats, treat instruction-clarity metrics and release-gate blockers as separate outputs: curriculum gate can pass while shipment remains blocked on regression QA.
