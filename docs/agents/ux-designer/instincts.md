# UX Designer — Instincts

Atomic behavioral patterns with confidence scores.
Higher confidence = more reliable pattern.

- **[0.9]** Apply the 3-item rule: never present more than 3 simultaneous choices for ages 3–5 — on every screen design
  - Source: NNG research + Piaget working memory limits (3–4 items in preoperational stage)

- **[0.9]** One purpose per screen — each child-facing screen should ask the child to do exactly one thing
  - Source: Medium "UI/UX Guidelines for Ages 3–6" + NNG guidelines

- **[0.9]** Audio-first: if removing all text makes a child-facing screen unusable, the design has failed — on every child interaction
  - Source: NNG "Children's UX" report, Dubiland project rule

- **[0.8]** First interaction must guarantee success — on onboarding and new game introduction
  - Source: NNG FTUE research + scaffolding literature (ERIC EJ1286593)

- **[0.8]** Every tap must produce feedback within 100ms (visual + audio) — on all interactive elements
  - Source: Gelman's Response principle + Duolingo ABC benchmark

- **[0.8]** Scaffold down after 2 failures, scaffold up after 3 successes — on difficulty progression design
  - Source: ZPD/scaffolding research, "Mobile App Features That Scaffold Pre-School Learning" study

- **[0.8]** דובי's gaze should direct attention to the primary action area — on every screen with the mascot
  - Source: Character-driven UX research, Khan Academy Kids benchmark

- **[0.7]** Design two evaluation passes: parent 3-second trust test + child "what would I tap" test — on every new layout
  - Source: Two-user problem analysis from Cantuni's "Designing Digital Products for Kids"

- **[0.7]** Use slow entrances (300–500ms) and spring/bounce for celebrations — on all motion design
  - Source: NNG physical development guidelines — kids need time to track motion

- **[0.7]** Present repetition as varied play, not drill — on game and activity design
  - Source: Gelman's Investment principle + spaced repetition research

- **[0.6]** Compare every new design against Khan Academy Kids, Duolingo ABC, and Toca Boca — on major design decisions
  - Source: Benchmark analysis of leading children's educational apps
