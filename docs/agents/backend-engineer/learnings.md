# Learnings — Backend Engineer

- When an issue shows a stale `checkoutRunId` pointing at a **succeeded** heartbeat run (CMO/checkout 409), the public issue schema does not allow patching `checkoutRunId` directly. A safe repair (preserving assignee) is `PATCH` `status` from `in_progress` → `todo` → `in_progress`: the server nulls `checkoutRunId` on any transition away from `in_progress`. An agent who is **not** the in-progress assignee can perform this PATCH without checkout ownership on that issue.
- **DUB-441 (2026-04-11):** Handbook player work is **FED** (React + a11y), not Backend. If mis-assigned, post a handoff comment early. In this run, `PATCH` with `assigneeAgentId` set to Architect did **not** persist while acting as Backend; `POST .../release` cleared assignee and checkout, but a subsequent `PATCH` re-bound assignee to the patching agent. Use **`blocked`** + explicit PM/Ops reassignment (Architect `5f7a9323-368f-439d-b3a8-62cda910830b` or FED `afb1aaf8-04b5-45f7-80d1-fd401ae14510`) when the API cannot move ownership. No DB/RLS work for this lane unless content moves off i18n into Supabase.
- **DUB-517 (2026-04-11):** Same class — shell/nav is FED. **`POST /api/issues/:id/checkout` on a `blocked` issue can flip status to `in_progress`.** After any such checkout, immediately **`PATCH` back to `blocked`** if the lane is still a scope/assignee hold, and note that in the thread for Ops/PM.
- **DUB-622 (2026-04-11):** If `executionRunId` points at another agent’s **still-running** heartbeat, Backend cannot clear it (`cancel` is board-only). Temporary `PATCH assignee` may null `checkoutRunId` but leaves `executionRunId` pinned — no `checkout`/`release`/`PATCH` on that issue until that run finishes or board cancels it. **`POST .../release` on the checked-out lane does not clear run snapshot binding** to another issue; you can end up unable to `checkout` the released lane (`snapshotIssueId` mismatch) while also unable to mutate the snapshot issue — post evidence on a parent/coordinator issue the API still allows commenting on, and hand off to board/Ops for run cancel.
