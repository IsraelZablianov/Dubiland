# Letter Storybook — Association Image Style and Prompt Pipeline

- Source issue: [DUB-655](/DUB/issues/DUB-655)
- Parent initiative: [DUB-647](/DUB/issues/DUB-647)
- Scope: image-generation contract for "הסיפור של האותיות" association art (Hebrew letters + object scenes)

## 1) Delivery Contract

- Produce one primary association image per base Hebrew letter (`22` total).
- Produce five optional final-form bonus images (`ך`, `ם`, `ן`, `ף`, `ץ`) when PM confirms bonus-page scope.
- Keep all illustrations text-free: no baked words, no glyphs, no numerals, no watermark overlays.
- Keep composition audio-first friendly: image supports narration, never replaces it.

## 2) Visual Style Contract (Must Stay Stable Across All Letters)

### Art direction
- Premium children storybook look: watercolor + soft gouache texture.
- Warm, optimistic lighting; low visual clutter; one clear focal object.
- Soft but readable contrast for tablet screens.

### Dubi consistency
- דובי appears as a warm brown teddy with friendly expression.
- Small blue backpack remains visible whenever דובי is in frame.
- דובי should guide/point, not dominate the frame over the association object.

### Composition and RTL behavior
- Keep visual flow right-to-left (focal cues can begin on right and move left).
- Leave clean negative space for runtime overlays (if needed) without hiding the object.
- No UI chrome in artwork (no buttons, labels, speech bubbles).

## 3) Prompt Pipeline

### A. Input manifest (single source of truth before generation)

Create `docs/games/letter-storybook-association-manifest.csv` with these columns:

| Column | Meaning |
|---|---|
| `order` | Numeric storybook order (`01`-`22`, or `F1`-`F5` for final forms) |
| `letter` | Hebrew letter (`א` ... `ת`) |
| `slug` | Stable filename slug (from section 4 table) |
| `associationWordHe` | Approved Hebrew association word |
| `associationWordEn` | English gloss for prompt clarity |
| `sceneBriefHe` | One Hebrew sentence describing the scene |
| `sceneBriefEn` | English mirror of scene brief |
| `dubiRole` | `present` or `offscreen` |
| `ageBand` | `3-4`, `4-5`, `5-6`, or `6-7` |

Use this manifest to generate prompts deterministically and prevent per-batch drift.

### B. Global style prefix (prepend to every prompt)

```text
Children's educational storybook illustration for ages 3-7, watercolor and soft gouache texture, warm cinematic light, clean silhouettes, low clutter, one focal object, Hebrew RTL-aware composition, no UI elements.
If Dubi is present: warm brown teddy bear, friendly eyes, small blue backpack, supportive guide role.
Constraints: no text, no letters, no numerals, no logos, no watermarks, no uncanny faces, safe and gentle mood.
```

### C. Per-letter prompt template

```text
Create one 16:10 storybook illustration for a Hebrew letter-association page.
Target letter: {letter}
Association word in Hebrew: {associationWordHe}
Association meaning in English: {associationWordEn}
Scene brief (Hebrew): {sceneBriefHe}
Scene brief (English): {sceneBriefEn}
Dubi role: {dubiRole}
Age-band tone: {ageBand}
Composition requirements: clear focal object, medium shot, negative space for overlays, calm depth layers, right-to-left visual flow.
Output intent: single static background illustration for app storybook page.
```

### D. Calibration batch policy

- First generate a 4-letter pilot batch: `01`, `02`, `03`, `04`.
- Run style QA (section 6) and approve before generating the remaining letters.
- Keep one prompt template version string in comments (`v1`, `v2`) so iterations stay traceable.

- Base letters: `letter-{order}-{slug}.jpg` in `assets-src`
- Final forms: `letter-final-{slug}.jpg` in `assets-src`

## 6) QA and Performance Gate

Before handoff to FED/PM:

1. Visual checks:
- No text or glyph artifacts.
- Association object is obvious in under 1 second.
- דובי consistency (fur tone, backpack, face shape) remains stable.
- RTL directional cues feel natural.

2. Pipeline checks:
- Run `yarn assets:pipeline` to generate optimized variants.
- Confirm manifest entries are present in `packages/web/public/images-manifest.json`.
- Run `yarn workspace @dubiland/web images:budgets`.

3. Spot review in product shell:
- Validate at least one mobile viewport (`390x844`) and one tablet viewport (`768x1024`).

## 7) Execution Risks and Mitigations

| Risk | Impact | Mitigation | Owner |
|---|---|---|---|
| Association word changed after image generation | Rework and path churn | Keep filenames letter-based only; update manifest words without renaming assets | PM + Content Writer |
| Style drift across batches | Inconsistent storybook look | Lock global prefix + template version; approve 4-letter pilot first | Media Expert |
| Model adds accidental text/letter marks | Violates literacy UX contract | Hard negative constraints in every prompt; reject and regenerate immediately | Media Expert |
| דובי visual inconsistency between letters | Brand mismatch | Enforce Dubi identity checklist in QA; regenerate outliers in same batch | Media Expert |
| Overweight assets fail web budget gates | Build and runtime regression | Route all files through `assets-src` + `yarn assets:pipeline` + budgets check | Media Expert + FED |
| Board-proxy turnaround delays | Storybook spec blocked waiting on assets | Ship prompts in small bounded batches (4-6 images) with exact paths | Media Expert + PM |
| Ambiguous associations for younger ages | Lower comprehension | Validate association list with Reading PM + Children Learning PM before full render | PM + Reading PM + Children Learning PM |

## 8) Handoff Expectations

- Reading PM can reference this file directly when finalizing `docs/games/letter-storybook.md`.
- Content Writer owns final `associationWordHe` choices and narration text.
- Media Expert owns prompt execution, QA gate, and batch-by-batch evidence comments.
